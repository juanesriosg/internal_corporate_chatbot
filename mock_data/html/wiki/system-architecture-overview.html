Architecture Style

The company uses a service-oriented architecture for customer-facing systems and event-driven workflows for asynchronous processing.

Synchronous APIs should have explicit timeouts and bounded retries. Asynchronous jobs should be idempotent.

Core Components

  • API Gateway routes external traffic to service APIs.
  • Event Bus distributes domain events such as AccountCreated and InvoicePaid.
  • Data Platform stores analytical copies of events for reporting.
  • Knowledge Assistant uses ingestion, retrieval, orchestration, and evaluation components.

Design Rules

Services own their databases. Shared database access across services is prohibited. Data contracts should be versioned and documented before breaking changes.
